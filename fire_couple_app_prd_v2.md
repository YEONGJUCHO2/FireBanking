## 0. 문서 목적

이 문서는 기존 v1 기획을 **클로즈드 베타 구현 가능한 수준으로 재정리한 PRD**다.

이 문서가 제품 범위의 단일 출처다. `docs/superpowers/specs/*`, `docs/superpowers/plans/*`, `.gstack/*`, `.superpowers/*`에 있는 내용은 구현/논의 히스토리로만 본다. 서로 충돌하면 이 PRD가 우선한다.

핵심은 세 가지다.

1. 제품 정체성을 흔들리지 않게 고정한다.
2. MVP 범위를 분명히 잘라 개발 중 과욕을 막는다.
3. FIRE 생활비 조정기를 부가기능이 아니라 **FIRE 계산의 현금흐름 엔진**으로 재정의한다.

### 0-1. 2026-05-03 클로즈드 베타 보정 결정

이번 배포는 완전 공개 출시가 아니라 지인/초기 커플 몇 팀을 대상으로 하는 클로즈드 베타다. 목표는 많은 기능을 보여주는 것이 아니라, 사용자가 재무 데이터를 입력해도 될 만큼 신뢰할 수 있는 첫 루프를 완성하는 것이다.

클로즈드 베타의 북극성:

```text
리드 파트너가 1분 안에 첫 FIRE 결과를 보고,
배우자에게 보여줄 만하다고 느끼게 만든다.
```

이번 베타에서 반드시 반영할 보정:

- Admin 온보딩은 4개 숫자만 먼저 받는다.
- 온보딩 완료 직후 대시보드가 아니라 `첫 FIRE 결과 브릿지`를 보여준다.
- 배우자 초대는 결과 브릿지에서 강하게 추천하되, 대시보드 먼저 보기도 허용한다.
- 대시보드의 배우자 카드는 미완료 상태에서만 노출한다.
- 하단 탭은 `홈 / 기록 / 함께 / 설정` 4개로 정리한다.
- 별도 `분석` 탭은 클로즈드 베타에서 숨기고, 필요한 분석은 홈/기록에 흡수한다.
- FIRE 생활비 조정기는 `고정비 / 변동비 / 버퍼` 3개 묶음을 먼저 보여준다.
- FIRE 자산 진단은 종목 검색보다 `FIRE 반영 순자산 공식`을 먼저 보여준다.
- 설정 화면은 실제 동작하는 항목만 남긴다.
- 하드코딩된 데모 이름, 기록, 분석 숫자는 실제 데이터 또는 빈 상태로 바꾼다.
- 온보딩, 홈, 기록, 공유, 자산 진단은 하나의 공용 FIRE 계산 모델만 사용한다.
- 결과/자산/설정에는 25배 룰, 연 5%, 투자 자문 아님, 부부 공유 범위를 명확히 노출한다.

## 1. 제품 한 줄 정의

**월 1회 3분, 부부가 함께 순자산과 경제적 자유(FIRE) 진척을 확인하는 앱**

보조 정의:
- 부부/커플이 같은 목표를 보고 돈 대화를 덜 불편하게 하도록 돕는 앱
- 가계부가 아니라 **공동 자산 & 현금흐름 & FIRE 진척 관리 도구**
- 자동 연동이 아니라 **월 단위 스냅샷 기반 리추얼 앱**

---

## 2. 해결하려는 문제

### 2-1. 사용자 문제

부부 중 한 명은 FIRE, 저축, 투자, 자산 형성에 진심이지만 다른 한 명은 상대적으로 관심이 적다.
기존 앱은 아래 둘 중 하나로 쏠린다.

- 개인용 FIRE 계산기: 혼자 계산하기엔 좋지만 부부 공동 관리가 안 됨
- 공동 가계부: 함께 쓰긴 좋지만 FIRE 목표 관리에는 약함

결과적으로 생기는 문제:

- 한쪽은 혼자 재무 목표를 끌고 가느라 답답함
- 두 사람이 같은 숫자를 보지 않아 돈 대화가 반복적으로 꼬임
- 가계부는 귀찮고, FIRE는 개인 중심이라 **부부용 중간 지점이 비어 있음**

### 2-2. 우리가 만드는 해법

- 월 1회 체크인으로 입력 부담을 낮춘다.
- Admin/Lite 비대칭 UX로 참여 장벽을 낮춘다.
- 고정비/현금흐름과 순자산을 FIRE 목표 계산에 연결한다.
- “이번 달 덕분에 FIRE가 얼마나 앞당겨졌는지”를 보여준다.

---

## 3. 타겟 사용자

### 3-1. 1차 타겟

다음 조건을 만족하는 부부/예비부부/동거 커플.

1. 둘 중 한 명은 FIRE/조기은퇴/자산 성장에 강한 관심이 있다.
2. 다른 한 명은 돈 관리에 깊게 관여하진 않지만 완전히 거부하지는 않는다.
3. 월 1회 정도의 수동 입력은 감수할 수 있다.
4. 재무 데이터를 부부 공동 워크스페이스에서 함께 보는 것에 거부감이 적다.

### 3-2. 핵심 페르소나

#### Admin
- FIRE 관심자
- 자산, 대출, 투자, 목표 설정에 적극적
- 엑셀/계산/시뮬레이션을 좋아함
- 배우자와 같은 목표를 보고 싶어함

#### Lite
- 돈 관리에 큰 재미는 없음
- 복잡한 입력은 싫어함
- 하지만 관계를 위해 최소 참여는 가능함
- “큰 숫자 1개 + 버튼 1개” 수준의 UX를 선호함

## 4. 제품 원칙

### 원칙 1. 월 체크인이 기본이다
- 매일 입력하는 가계부가 아니다.
- 월 1회 체크인과 지난달 값 재사용이 기본이다.

### 원칙 2. 입력 부담은 비대칭이어야 한다
- Admin은 자세히 입력할 수 있다.
- Lite는 최소 입력으로도 참여할 수 있어야 한다.

### 원칙 3. FIRE 생활비 조정기는 핵심이다
- 고정비/변동비/버퍼는 부가 기능이 아니다.
- FIRE 목표 생활비와 예상 도달 시점을 떠받치는 핵심 엔진이다.

### 원칙 4. 100% 공유 원칙을 유지한다
- 부부 워크스페이스 안의 금액 정보는 공동으로 본다.
- 단, 계좌번호/상세 거래내역/금융 로그인 정보 같은 민감 원본 데이터는 애초에 수집하지 않는다.

### 원칙 5. 자동화보다 관계형 리추얼이 우선이다
- 마이데이터 자동 연동을 지향하지 않는다.
- 대신 월간 체크인, 배우자 초대/Lite 입력, 공유 리포트로 반복 사용을 만든다.

### 원칙 6. 결과는 미래 중심으로 보여준다
- 과거 소비를 추궁하지 않는다.
- “이번 달 덕분에 목표가 얼마나 앞당겨졌는지”를 보여준다.

---

## 5. 이 앱이 아닌 것

- 상세 소비 가계부 앱이 아니다.
- 마이데이터 연동 앱이 아니다.
- 투자 추천 앱이 아니다.
- 세전 월급/세금 계산 중심의 급여 계산기가 아니다.

이 앱은 **부부 공동 자산/현금흐름/FIRE 진척 관리 앱**이다.

### 7-3. 이후 릴리즈 후보

아래 기능은 제품 방향에는 맞지만 클로즈드 베타의 신뢰 루프가 먼저 검증된 뒤 넣는다.

1. Request 시스템 (베타 이후)
2. 상세 알림
3. 기록 상세 분석
4. 고급 FIRE 설정
5. 연금/퇴직금/세금/인플레이션 모델링

### 7-4. 왜 이렇게 자르나

- 먼저 확인할 것은 “부부가 같은 FIRE 결과를 보고 대화할 수 있는가”다.
- 클로즈드 베타에서는 가짜 기능, 하드코딩 데이터, 서로 다른 계산 결과가 가장 큰 신뢰 리스크다.
- 고정비 브랜치와 자산 진단은 핵심이지만, 첫 화면은 단순해야 한다.
- AI/CSV/포트폴리오 분석은 개발량 대비 베타 검증 가치가 낮다.

## 8. 정보 구조 (IA)

클로즈드 베타 하단 탭은 4개만 둔다.

```text
홈 / 대시보드
├─ 이번 달 FIRE 결과
├─ 목표 월 생활비와 FIRE 목표자산
├─ FIRE 반영 순자산
├─ 예상 도달 시점
├─ 이번 달 현금흐름
├─ 미완료 상태의 배우자 체크인 카드
├─ FIRE 생활비 조정기 진입
└─ FIRE 자산 진단 진입

기록
├─ 월별 스냅샷 목록
├─ 지난달 대비 변화
├─ 확정/임시 상태
└─ 데이터 없음 상태

함께
├─ 배우자 연결 상태
├─ 이번 달 체크인 상태
├─ 초대 링크 재공유
└─ Lite 입력 상태

설정
├─ 프로필
├─ 데이터/공유 범위
├─ 계산 가정과 면책
└─ 로그아웃
```

별도 `분석` 탭은 클로즈드 베타에서 숨긴다. 저축률, 지출 분포, 지난달 대비 변화처럼 필요한 분석은 홈 카드나 기록 상세로 흡수한다.

보조 화면:

```text
온보딩
└─ 4개 숫자 빠른 입력

첫 FIRE 결과 브릿지
├─ FIRE 기준 설명
├─ 남은 금액/기간
├─ 배우자 초대 CTA
└─ 대시보드 먼저 보기

FIRE 생활비 조정기
├─ 고정비
├─ 변동비
├─ 버퍼
└─ 세부 항목 조정

FIRE 자산 진단
├─ FIRE 반영 순자산 공식
├─ 투자자산
├─ 제한/미래 자산
└─ 투자 연동 대출
```

## 9. 핵심 사용자 플로우

## 9-1. Admin 온보딩

```text
1. 회원가입
2. 4개 숫자 빠른 입력
   - 목표 월 생활비
   - 세후 월수입
   - 월 총지출
   - 투자가능 순자산
3. 첫 FIRE 결과 브릿지 표시
4. 배우자에게 3개 숫자 부탁하기 또는 대시보드 먼저 보기
5. 선택: FIRE 생활비 조정기 또는 FIRE 자산 진단으로 이동
```

### 목적
- 리드 파트너가 1분 안에 첫 FIRE 결과를 본다.
- "이 정도면 배우자에게 보여줄 수 있겠다"는 확신을 만든다.
- 상세 생활비/자산 입력은 첫 결과 이후 선택 흐름으로 둔다.

### 첫 FIRE 결과 브릿지

온보딩 완료 후 바로 대시보드로 보내지 않는다. 먼저 아래 목적의 단일 화면을 보여준다.

```text
월 300만원으로 살려면 FIRE 기준은 9억원이에요.
현재 투자가능 순자산 기준으로는 7억 6,500만원 남았습니다.
배우자 숫자 3개가 더해지면 우리 가족 기준으로 더 정확해져요.
```

CTA:

- Primary: `배우자에게 3개 숫자 부탁하기`
- Secondary: `대시보드 먼저 보기`

이 화면은 대시보드를 대체하지 않는다. 첫 결과를 이해시키고 초대 의향을 만드는 브릿지다.

## 9-2. Lite 온보딩

```text
1. 초대 링크 접속
2. 회원가입
3. "OO님이 당신을 초대했어요" 화면
4. 내 세후 월수입 입력
5. 내 월 반복지출 총액 입력
6. 내 월 정기저축/투자 총액 입력
7. 완료
```

### Lite 원칙
- 총액 중심으로 입력한다.
- 상세 카테고리는 선택 입력이다.
- 이후 월 체크인에서는 "지난달과 같아요"가 기본이다.
- 개별 입력값의 공유 범위를 명확히 설명한다.

## 9-3. 월 체크인

### Admin
```text
- 자산/부채 월말 스냅샷 확인
- 지난달 값 불러오기
- 변경분만 수정
- 고정비/변동비/투자값 확인
- 제출
```

### Lite
```text
- 이번 달 세후 수입 확인/수정
- 반복지출 지난달과 같음 여부 확인
- 정기저축/투자 지난달과 같음 여부 확인
- 큰 변경 있으면 총액 수정
- 제출
```

## 9-4. 체크인 완료 후 결과

둘 다 완료되면 월간 리포트를 보여준다.

표시 항목:
- 부부 총 순자산
- 지난달 대비 증감
- 월 자산 증가 여력
- FIRE 목표 자산
- 예상 도달 시점
- 지난 체크인 대비 얼마나 앞당겨졌는지

### 중요 결정
- **부분 결과는 보여준다.**
- 배우자 미완료 상태에서도 임시 결과를 보여주되, 최종 리포트는 둘 다 완료해야 확정된다.

예시:
```text
현재 입력 기준 예상 도달일: 2034년 8월
배우자 체크인이 완료되면 이번 달 결과가 확정돼요.
```

## 11. 배우자 초대 / Lite 체크인 루프

### 목적
클로즈드 베타에서는 별도 Request 시스템을 만들지 않는다.
대신 Admin이 결과 브릿지에서 배우자를 초대하고, Lite는 링크를 통해 3개 숫자만 입력해 월간 결과를 확정한다.

### MVP 동작
1. Admin 온보딩 완료 후 첫 FIRE 결과 브릿지에서 초대 CTA를 보여준다.
2. Admin이 초대 링크를 복사하거나 공유한다.
3. Lite는 링크에서 본인 세후 월수입, 반복지출, 정기저축/투자 총액만 입력한다.
4. Lite 입력이 끝나면 Admin 홈의 배우자 상태 카드가 완료 상태로 바뀐다.
5. 둘 다 입력되면 이번 달 결과를 확정하고 히스토리에 남긴다.

### UI 원칙
- Lite 화면은 숫자 3개와 확인 버튼 중심으로 끝낸다.
- Lite에게 FIRE 이론을 길게 설명하지 않는다.
- 입력 완료 후에는 "덕분에 이번 달 결과가 더 정확해졌다"는 영향 피드백을 보여준다.

예시:
```text
입력 완료
덕분에 이번 달 FIRE 예상일이 더 정확해졌어요.
```

## 12. FIRE 생활비 조정기 / 월 현금흐름 브랜치

## 12-1. 최종 정의

FIRE 생활비 조정기는 별도 부가기능이 아니다.

**월 현금흐름을 계산하고, 그 결과를 FIRE 목표 계산에 넘겨주는 핵심 엔진**이다.

제품 내 표기 권장:
- 사용자용 이름: `FIRE 생활비 조정기`
- 내부 개발 모듈명: `cashflow`
- 설명 문구: `생활비를 정리하면 FIRE 날짜가 더 정확해져요`

## 12-2. 브랜치가 다루는 입력값

### 공통 입력
- 세후 월수입
- 고정생활비
- 변동비 예산
- 정기저축/투자

### 상세 입력
- 주거비
- 관리비
- 보험료
- 통신비
- 구독비
- 교통비
- 기타 고정비
- 대출 원금상환
- 대출 이자

## 12-3. 왜 넣는가

1. FIRE 목표 자산은 연지출을 알아야 계산 가능하다.
2. FIRE 도달일은 월 자산 증가 속도를 알아야 계산 가능하다.
3. 고정비 절감은 사용자가 체감하기 쉬운 행동 변화 포인트다.

## 12-4. 핵심 원칙

### 원칙 1. 한번 세팅하면 재사용한다
- 월별 체크인 때 지난달 값을 기본값으로 불러온다.
- 대부분 사용자는 큰 변경 없을 경우 그대로 제출 가능해야 한다.

### 원칙 2. 실제값과 시뮬레이션값을 분리한다
- 실제 대시보드에 반영되는 값과
- “만약 줄인다면?” 실험용 값은 분리한다.

### 원칙 3. 현재 지출과 은퇴 후 지출을 분리한다
- 모든 현재 지출이 FIRE 목표 연지출에 그대로 들어가면 안 된다.

예:
- 대출 원금상환: 현재 현금흐름에는 포함, FIRE 연지출에는 제외
- 정기투자: 현재 현금흐름에는 포함, FIRE 연지출에는 제외
- 대출 이자: 현재 현금흐름에는 포함, 상황에 따라 FIRE 연지출 포함 여부 선택

## 13. 계산 엔진 정의

## 13-1. 핵심 용어

### 부부 총 세후수입
```text
Admin 세후 월수입 + Lite 세후 월수입
```

### 월 생활비
FIRE 계산에 쓰는 현재 생활비.

```text
고정비 중 FIRE 연지출 포함 항목
+ 변동비 예산
+ 대출 이자(포함 설정 시)
```

### 연 생활비
```text
월 생활비 × 12
```

### FIRE 목표 자산
```text
연 생활비 × 25
```

### 남는 현금
```text
부부 총 세후수입
- 현재 현금흐름 포함 고정비
- 변동비
- 대출 원금상환
- 정기저축/투자
```

### 월 자산 증가 여력
```text
정기저축/투자
+ 대출 원금상환
+ 남는 현금
```

> 남는 현금이 음수일 수 있다. 이 경우 월 자산 증가 여력은 자동으로 줄어든다.

## 13-2. 기본 예시

```text
부부 총 세후수입: 720만원
고정생활비: 230만원
변동비: 170만원
대출 원금상환: 80만원
대출 이자: 40만원
정기투자: 200만원
```

### 월 생활비
```text
230 + 170 + 40 = 440만원
```

### 연 생활비
```text
440 × 12 = 5,280만원
```

### FIRE 목표 자산
```text
5,280만원 × 25 = 13억 2,000만원
```

### 남는 현금
```text
720 - 230 - 170 - 80 - 40 - 200 = 0만원
```

### 월 자산 증가 여력
```text
200 + 80 + 0 = 280만원
```

## 13-3. 예상 도달 시점 계산

MVP에서는 아래 규칙으로 간다.

- 현재 순자산
- 목표 자산
- 월 자산 증가 여력
- 기본 기대 수익률 5% 연복리

를 기준으로 **월 단위 시뮬레이션**을 돌려 목표 도달 월을 계산한다.

### MVP 규칙
- 사용자는 수익률을 직접 조절하지 않는다.
- 기본 가정은 시스템 기본값으로 둔다.
- 고급 모드(수익률/인플레이션 조절)는 후속 버전에서 제공한다.

## 13-4. 고정비 절감 효과 계산

예시:
```text
구독비 5만원 감소
→ 월 생활비 5만원 감소
→ FIRE 목표 자산 감소
→ 월 남는 현금 증가
→ 예상 도달일 재계산
→ “약 2개월 앞당겨질 수 있어요” 표시
```

## 14. 실제값 vs 시뮬레이션값

## 14-1. 실제값 모드

대시보드, 월간 리포트, 공식 FIRE 목표에 반영되는 값.

예:
- 실제 통신비 12만원
- 실제 구독비 8만원
- 실제 정기투자 200만원

## 14-2. 시뮬레이션 모드

사용자가 가정값을 바꿔보는 미리보기 모드.

예:
- 구독비를 8만원 → 3만원으로 줄이면?
- 투자금을 200만원 → 250만원으로 늘리면?

### 중요 규칙
시뮬레이션 모드에서 바꾼 값은 자동 저장되지 않는다.

반드시 아래 버튼을 눌러야 실제값으로 반영된다.

```text
[이 계획을 실제 고정비에 반영하기]
```

## 18. 화면 정의

## 18-1. 홈 / 대시보드

표시 요소:
- 이번 달 FIRE까지 남은 금액/기간
- FIRE 목표 그래프
- 목표 월 생활비
- FIRE 목표 자산
- FIRE 반영 순자산
- 예상 도달 시점
- 이번 달 자산 증가 여력
- 체크인 상태
- 이번 달 현금흐름
- FIRE 생활비 조정기 진입
- FIRE 자산 진단 진입

배우자 관련 카드는 미완료 상태에서만 보여준다.

- 초대 전: 배우자 숫자 3개를 더하면 결과가 더 정확해진다는 작은 추천 카드
- 초대 보냄/수락 전: 대기 상태와 링크 다시 복사
- 수락했지만 입력 전: 입력 대기 상태와 부담 없는 리마인드
- 입력 완료: 초대 카드는 제거하고 이번 달 결과 확정 배지 또는 체크인 완료 상태만 표시

FIRE 목표 그래프 필수 조건:

- `FIRE까지 남은 금액` 제목, 남은 금액, 가로 진행 바, 현재 위치 마커, 0%/25%/50%/100% 눈금을 포함한다.
- 100% FIRE 막대 끝에는 현재 구현된 움직이는 불꽃 GIF를 유지한다.
- 현재 구현 위치는 `components/fire-banking/fire-timeline.tsx`의 `FireEmoji`이며, GIF 소스는 `https://em-content.zobj.net/source/animated-noto-color-emoji/356/fire_1f525.gif`다.
- 모바일과 데스크톱 모두에서 눈금, 금액, 불꽃 GIF가 겹치지 않아야 한다.
- 0%, 진행 중, 100%, 목표 초과 상태를 안정적으로 처리한다.

기록 화면은 하드코딩 샘플을 보여주지 않는다. 스냅샷이 없으면 "아직 저장된 월별 기록이 없어요"와 다음 행동을 보여준다.

## 18-4. 함께 화면

구성:
- 리드 파트너와 배우자 상태
- 초대/수락/입력 대기 상태
- 초대 링크 재공유
- 다음 체크인 예정일

배우자 관리 대부분은 홈이 아니라 함께 탭에서 다룬다. 홈은 미완료 상태의 할 일만 보여준다.

## 18-5. FIRE 자산 진단 화면

구성:
- FIRE 반영 순자산 공식 카드
- FIRE 반영 투자자산
- 제한/미래 자산
- 참고 자산
- 투자 연동 대출
- 종목 검색과 수동 입력

상단 공식:

```text
FIRE 반영 순자산
= 바로 쓸 수 있는 투자자산
- 투자 연동 대출
```

거주 부동산, 연금저축, IRP는 숨기지 않지만 기본 FIRE 순자산 계산에서 분리한다.

## 18-6. FIRE 생활비 조정기 화면

구성:
- 고정비
- 변동비
- 버퍼
- 추천 목표 월 생활비
- 대시보드 기준과의 차이
- 세부 항목 조정

첫 화면은 3개 묶음만 보여준다. 세부 항목은 `세부 항목으로 조정하기` 안에 접는다. 추천값 적용 시 상단 3개 숫자와 추천 목표 월 생활비가 즉시 바뀌어야 한다.

## 18-7. 설정 화면

구성:
- 프로필
- 데이터와 부부 공유 범위
- FIRE 계산 가정과 면책
- 로그아웃

실제로 동작하지 않는 CSV, 도움말, 보안 링크는 클로즈드 베타에서 숨긴다.

## 18-8. 별도 분석 화면

클로즈드 베타에서는 별도 분석 탭과 화면을 숨긴다. 대시보드와 다른 계산값을 보여주는 분석 화면은 신뢰 리스크가 크다. 필요한 분석 지표는 홈 또는 기록 상세에 흡수한다.

## 25. 리스크와 대응

## 리스크 1. 온보딩이 길어질 수 있음
대응:
- 첫 화면에서는 총액만 받는다.
- 첫 결과를 먼저 보여주고 상세 설정으로 유도한다.

## 리스크 2. Lite 이탈
대응:
- Lite는 세후 월수입, 반복지출, 정기저축/투자 총액 3개 입력 중심으로 설계한다.
- “지난달과 같아요”를 기본값으로 둔다.
- FIRE 설명은 짧게 유지하고 완료 후 영향 피드백을 먼저 준다.

## 리스크 3. 계산 신뢰도 부족
대응:
- 세전/세후 자동 계산을 하지 않는다.
- 실제 세후 월수입 직접 입력을 기본으로 한다.
- FIRE 결과는 “예상”으로 표현한다.

## 리스크 4. 고정비 항목이 너무 많아질 수 있음
대응:
- 첫 화면은 고정비/변동비/버퍼 3개 묶음만 제공한다.
- 세부 항목은 접힌 조정 화면으로 보낸다.
- 추천값 적용 시 상단 합계가 바로 변하도록 한다.

## 리스크 5. 결과 잠금으로 인한 짜증
대응:
- 부분 결과는 보여주고 최종 결과만 확정 잠금 처리한다.

## 리스크 6. 데모 데이터가 실제 기능처럼 보일 수 있음
대응:
- 하드코딩된 이름, 기록, 분석 숫자는 제거한다.
- 실제 데이터가 없으면 빈 상태와 다음 행동을 보여준다.

## 리스크 7. 가짜 설정/분석 기능이 신뢰를 깎을 수 있음
대응:
- 별도 분석 탭은 클로즈드 베타에서 숨긴다.
- 설정 화면은 실제 동작하는 항목만 노출한다.
- CSV/도움말/보안처럼 구현되지 않은 행은 숨긴다.

## 리스크 8. 화면마다 FIRE 숫자가 다를 수 있음
대응:
- 온보딩 결과, 홈, 기록, 공유, 자산 진단은 하나의 공용 FIRE 계산 모델만 사용한다.
- 계산 불가 상태는 0년 0개월처럼 보정하지 않고 "계산하기 어려움"으로 표현한다.

- FIRE 자산 진단은 FIRE 반영 순자산 공식을 먼저 보여준다.

## 29. 2026-04-25 리뷰 반영 결정

이 섹션은 MVP 설계와 R0 구현계획에서 우선 적용할 보정 결정이다. 기존 PRD의 방향은 유지하되, 아래 항목은 이후 문서와 구현에서 우선한다.

### R0의 위치

R0 단독은 외부 MVP가 아니다. R0는 내부 알파다.

R0가 검증하는 것:

- 리드 파트너가 첫 FIRE 결과를 볼 수 있는가
- 입력값과 계산 가정을 신뢰할 수 있는가
- 저장된 월간 스냅샷을 다시 볼 수 있는가
- 배우자 초대 링크를 만들 의향이 있는가

R0가 검증하지 않는 것:

- 부부가 실제로 함께 체크인하는가
- 배우자가 입력을 완료하는가
- 월 1회 리추얼이 반복되는가
- Admin/Lite 비대칭 UX가 관계적으로 안전한가

첫 외부 검증 단위는 R0+R1로 본다.

### 명칭 원칙

DB나 내부 권한명으로 `admin`, `lite`를 쓸 수는 있다. 하지만 사용자 화면에서는 위계감을 줄 수 있는 "Admin", "Lite", "관리자" 표현을 기본 노출하지 않는다.

사용자-facing 표현은 아래처럼 중립화한다.

- 리드 파트너
- 배우자
- 함께 보기
- 체크인 완료/대기
- 이름 기반 표시

### 한국형 FIRE 입력 원칙

순자산을 하나의 숫자로만 받지 않는다. R0부터 최소 세 가지로 나눈다.

- 투자가능 순자산: FIRE 계산에 포함
- 거주 부동산 순자산: 표시하되 R0 FIRE 계산에서는 제외
- 기타 순자산: 표시하되 R0 FIRE 계산에서는 제외

자가 부동산을 FIRE 계산에 그대로 넣으면 한국 사용자에게 예상 도달일이 과도하게 낙관적으로 보일 수 있다. 결과 신뢰도를 해치는 핵심 위험이므로 R0에서 바로 분리한다.

### FIRE 가정과 면책

R0는 단순성을 위해 연 5% 수익률과 25배 룰을 사용한다. 대신 결과 화면에 이 가정을 반드시 노출한다.

필수 노출 문구 성격:

```text
연 5%, 25배 룰 기준의 참고용 시뮬레이션이에요.
거주 부동산, 연금 수령 시점, 세금, 인플레이션은 단순화되어 있어요.
투자 자문이 아닙니다.
```

연금, 퇴직금, IRP, 국민연금은 한국 FIRE 계산에서 중요하다. 다만 R0 계산식에는 넣지 않고, R2 이후 고급 FIRE 설정 또는 은퇴 이후 현금흐름 모델에서 다룬다.

### 비용 입력 정의

R0의 월 변동비는 "지난달 실제 지출"이 아니라 "평소 한 달 기준 예상 변동비"로 정의한다.

R1 전에 부부 합산 규칙을 확정한다. 기본 방향은 다음과 같다.

- 고정비와 변동비는 가구 단위로 리드 파트너가 입력한다.
- 개인별 소득과 정기저축/투자는 각자 입력값을 합산한다.
- 중복 입력을 막기 위해 화면에서 "가구 단위"와 "개인 단위"를 명확히 구분한다.
