# FIRE 생활비 조정기 설계

> 문서 상태: historical implementation note. 현재 제품 범위와 우선순위는 `fire_couple_app_prd_v2.md`가 단일 출처다. 이 문서는 FIRE 생활비 조정기의 구현 맥락을 보존하기 위해 남긴다.

작성일: 2026-05-03

## 배경

현재 `/subscribe`의 고정비 시뮬레이터는 항목별 고정비와 변동비를 입력받지만, 대시보드의 FIRE 기준값과 직접 연결되지 않는다. 저장은 `fixed_cost_simulations`에 남지만 대시보드가 이를 읽지 않고, 시뮬레이터도 저장된 값을 다시 불러오지 않는다.

이 때문에 사용자는 계산기는 존재하지만 앱의 핵심 목표인 `목표 월 생활비`, `FIRE 목표자산`, `남은 금액`, `남은 기간`과 따로 논다고 느낀다.

## 제품 원칙

Fire Banking은 자산관리 앱이 아니라 FIRE 의사결정 앱이다. 따라서 생활비 입력 도구는 단순 지출 관리가 아니라 FIRE 목표를 더 정확히 정하기 위한 보조 도구여야 한다.

사용자가 직접 확정하는 값:

- FIRE 목표 월 생활비
- 현재 투자가능 순자산
- 월 자산 증가 여력

시뮬레이터가 도와주는 값:

- 현재 또는 목표 고정비
- 평균 변동비
- 여유 버퍼
- 추천 목표 월 생활비
- 추천 월 자산 증가 여력 변화

시뮬레이터 결과는 자동으로 대시보드를 덮어쓰지 않는다. 사용자가 `추천값 적용`을 눌렀을 때만 FIRE 기준값에 반영한다.

## 이름과 위치

기존 `고정비 시뮬레이터`는 `FIRE 생활비 조정기`로 변경한다.

이름 변경 이유:

- 변동비와 버퍼까지 포함해야 FIRE 목표 월 생활비를 계산할 수 있다.
- 고정비만 줄인다고 사용자의 은퇴 후 생활비가 결정되지는 않는다.
- 기능의 목적은 구독비 절감이 아니라 FIRE 기준값 조정이다.

대시보드 진입 링크도 `고정비 시뮬레이터`가 아니라 `FIRE 생활비 조정기`로 표시한다.

## 사용자 흐름

### Lite 기본 흐름

일반 사용자는 온보딩 또는 설정에서 아래 값을 직접 입력한다.

- 목표 월 생활비
- 투자가능 순자산
- 월 자산 증가 여력

이 흐름은 빠르고 가벼워야 한다. 사용자가 시뮬레이터를 쓰지 않아도 대시보드가 정상 작동해야 한다.

### Hard 사용자 정밀 흐름

정밀하게 계산하고 싶은 사용자는 `FIRE 생활비 조정기`로 들어간다.

조정기에서 사용자는 아래 값을 구성한다.

- 고정비: 주거, 보험, 통신, 구독, 대출 이자처럼 반복되는 지출
- 변동비: 식비, 외식, 교통, 병원, 쇼핑, 취미처럼 매달 흔들리는 지출
- 버퍼: 예비비, 여행, 가족 지원, 예상 밖 비용처럼 목표 생활 수준에 포함할 여유분

조정기는 아래 추천값을 만든다.

```text
추천 목표 월 생활비 = 고정비 + 변동비 + 버퍼
```

그리고 현재 대시보드 기준과 비교해 아래 변화를 보여준다.

- 목표 월 생활비 변화
- FIRE 목표자산 변화
- 남은 금액 변화
- 예상 도달 기간 변화
- 월 자산 증가 여력 변화

사용자가 `추천값 적용`을 누르면 선택한 값만 대시보드 기준값에 반영한다.

## 고정비/변동비 UX

고정비와 변동비는 같은 화면 안에 두되 역할을 분리한다.

고정비 섹션:

- 반복 지출 최적화가 목적이다.
- 기본 항목은 포함/제외 토글을 제공한다.
- 사용자가 직접 추가한 항목은 수정/삭제할 수 있다.
- `+ 고정비 추가`는 섹션 상단에 노출한다.

변동비 섹션:

- 생활 수준 추정이 목적이다.
- 기본 항목은 식비, 외식/카페, 교통, 병원/약, 쇼핑/생활용품, 여행/취미, 기타로 둔다.
- 항목별 금액 수정과 직접 추가/삭제를 제공한다.
- 변동비는 절감 압박보다 평균 추정과 목표 생활 수준 선택을 강조한다.

버퍼 섹션:

- 금액 직접 입력을 기본으로 한다.
- 추천 프리셋은 `0%`, `5%`, `10%`, `15%`처럼 목표 생활비 대비 비율로 제공할 수 있다.

## 대시보드 연동

대시보드는 다음 값을 하나의 FIRE 기준 상태로 읽는다.

- `targetMonthlyExpense`
- `fireTargetAsset`
- `fireNetWorth`
- `monthlyAssetGrowthCapacity`

`FIRE 생활비 조정기`는 같은 계산식을 사용해 변경 전후를 비교한다.

`fireTargetAsset`은 아래 식으로 계산한다.

```text
fireTargetAsset = targetMonthlyExpense * 12 * 25
```

남은 금액은 아래 식으로 계산한다.

```text
remainingAmount = max(fireTargetAsset - fireNetWorth, 0)
```

남은 기간은 월 자산 증가 여력이 0보다 클 때만 계산한다. 0 이하이면 예상 도달 시점을 보여주지 않고, 먼저 월 자산 증가 여력을 확보해야 한다는 메시지를 보여준다.

## 저장 정책

시뮬레이터는 임시 계산 상태와 적용된 FIRE 기준값을 구분한다.

- 임시 계산 상태: 사용자가 조정 중인 고정비/변동비/버퍼
- 적용된 FIRE 기준값: 대시보드 계산에 실제 반영된 값

저장 후 다시 `/subscribe`에 들어오면 마지막 시뮬레이션 상태를 불러온다.

`추천값 적용`은 별도 액션으로 다룬다. 단순 저장만으로 대시보드 기준값을 바꾸지 않는다.

## 성공 기준

- 일반 사용자는 시뮬레이터 없이 직접 입력만으로 FIRE 대시보드를 사용할 수 있다.
- 정밀 사용자는 생활비 조정기를 통해 추천 목표 월 생활비를 만들 수 있다.
- 추천값 적용 전에는 대시보드가 바뀌지 않는다.
- 추천값 적용 후에는 대시보드의 목표 월 생활비, FIRE 목표자산, 남은 금액/기간이 같은 계산식으로 갱신된다.
- 고정비/변동비 항목 추가, 수정, 삭제가 모바일 화면에서 명확하게 동작한다.
